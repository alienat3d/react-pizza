19.3.0 Что такое «Outlet»? Представим, что у нас приложение, где, на какой бы страницы мы не были, у нас всегда есть Header и Sidebar с меню слева, а меняется у нас только большая центральная контентная часть, как в большинстве веб-приложений статистики или менеджмента времени. 
(https://reactrouter.com/api/components/Outlet) 
Так вот, мы можем создать некий шаблон сайта, где в <Outlet/> будут попадать динамически разные компонента, в зависимости от ссылки переданной браузеру. (Альтернатива этому во VueJS — <RouterView />)
19.3.1 Пример:
<script>
import {Route} from "react-router"; 

function Dashboard() {
    return (
        <div>   
            <Sidebar />
            <h1>Dashboard</h1>

            {/* Этот элемент отрендерит либо комп. «DashboardMessages», если адрес в браузере "website.com/messages", либо комп. «DashboardTasks», если адрес "website.com/tasks". */}
            <Outlet /> 
        </div>
    )
}

function App() {
    return (
        <Routes>
            <Route path="/" element={<Dashboard />}>
                <Route path="messages" element={<DashboardMessages />} />
                <Route path="tasks" element={<DashboardTasks />} />
            </Route>
        </Routes>
    )
}
</script>
